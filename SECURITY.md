# Security Policy

## Supported Versions

This project is in early setup. Security fixes apply to the latest `main` branch.

## Reporting a Vulnerability

Please report suspected vulnerabilities privately.

Recommended report contents:
- vulnerability type and impact
- affected files/components
- reproduction steps or proof of concept
- suggested mitigation (if known)

Do not open public issues for active vulnerabilities before coordinated disclosure.

## Data Handling Expectations

Given tax and PII sensitivity:
- never include real PAN/TAN or personal data in issues/PRs
- use masked or synthetic fixtures only
